2. Shared Template & Global CSS

As Student 1, I was responsible for creating the shared HTML template structure and the global CSS file used by all team members.

HTML Structure

The base template uses semantic HTML5 elements:

  • <header> — contains the site logo and navigation bar
  • <nav> — navigation links to all main pages
  • <main> — placeholder for page-specific content
  • <footer> — site-wide footer with Page Editor and Validation links

CSS Techniques

The global CSS uses CSS custom properties (:root variables) to define the colour palette, ensuring consistent colours site-wide. I used Flexbox for the header and footer layouts, and CSS Grid for card grids and the footer columns. A sticky header (position: sticky; top: 0) keeps navigation accessible while scrolling.

Design Rationale

I chose a deep forest green (#1a6b3c) as the primary colour to evoke nature and environmental themes aligned with SDG 13. The warm amber accent (#f0a500) creates contrast and highlights calls to action. The Space Grotesk typeface provides a modern, readable feel, while Playfair Display adds character to headings.

3. Home Page

HTML Structure

The Home Page uses the shared template, adding a hero <section>, a statistics strip <div>, a mission cards <section>, and an interactive gallery <section>. Card content is wrapped in <article> elements for semantic clarity. The gallery tiles use <a> elements for accessibility and SEO.

CSS Techniques

The page uses embedded CSS within a <style> block in the <head>. This embedded CSS covers five required rules: (1) hero section gradient background and animation; (2) CSS keyframe animation for the hero overlay; (3) gallery tile hover effects including image scale and overlay transitions; (4) statistics strip grid layout; and (5) highlight list styling with custom arrows.

Design Rationale

The full-width hero was chosen to immediately communicate the urgency of climate action. The statistics strip creates an emotional impact before the user reads the detail cards. The interactive gallery tiles provide visual entry points to each team member's content page, encouraging exploration.

4. Gallery Page

HTML Structure

The Gallery page uses the shared template. The <main> section contains a page hero and a thumbnail grid. Each gallery item is a <div class="thumb-card"> with a data-id attribute linking it to a JavaScript data array. A modal dialog is implemented using a fixed-position <div> with role="dialog" and aria-modal="true".

JavaScript Implementation

All gallery interactivity is implemented with JavaScript:

  • Hover detection: mouseenter and mouseleave event listeners set a data-hovered attribute; CSS transitions provide the visual response.
  • Click-to-expand: A click event listener calls openModal(id) which populates the modal with the correct data from the galleryData array.
  • Close/collapse: Close button, backdrop click, and Escape key all call closeModal().
  • User customisation: Colour scheme and font dropdowns use change listeners to call applyCustomisation(), which sets inline styles on the modal content area.

CSS Techniques

Embedded CSS handles the thumbnail card hover effects, modal animation (@keyframes modalIn), and the modal control bar styling. Transitions on transform and box-shadow give cards a lift effect. Page-specific colour is applied to the gallery hero banner.

5. Content Page — The Science of Climate Change

HTML Structure

The content page is divided into five sections using <section> elements, each with a unique id for internal anchor linking. A sticky sidebar navigation uses <aside> with anchor links. The two-column layout uses CSS Grid (grid-template-columns: 220px 1fr).

Internal Navigation

Internal links use href="#section-id" anchors that target id attributes on each section. scroll-margin-top offsets scrolling to account for the sticky header. The Go-to-Top button uses href="#top" and position: fixed CSS — no JavaScript required.

Images

Four contextual images are used, each with descriptive alt text and onerror fallbacks. Images have border-radius, box-shadow, and responsive width: 100% applied through page-specific CSS.

6. Challenges & Lessons Learned

Challenge 1: Modal Accessibility

Implementing an accessible modal was more complex than expected. I initially omitted aria-modal and focus management. After research I added aria-modal="true", role="dialog", and focus transfer to the close button on open, which resolves the issue for screen reader users.

Challenge 2: Sticky Navigation Offset

When using internal anchor links with a sticky header, sections were being hidden behind the header. I resolved this by applying scroll-margin-top: 90px to each .article-section, which offsets the scroll target by the header height.

Challenge 3: Team CSS Coordination

Distributing the global CSS to team members and ensuring nobody accidentally overrode shared styles required clear communication. I added HTML comments marking which sections must not be modified, and a shared design brief document.

What I Learned

This project deepened my understanding of semantic HTML, CSS custom properties, and how JavaScript event delegation works. I also learned the importance of planning folder structure and naming conventions early in a group project to avoid broken links.

7. Accessibility Considerations

Accessibility was considered throughout development, in line with WCAG 2.1 guidelines:

  • Alternative text: All images include descriptive alt attributes. Decorative images use alt="".
  • Semantic HTML: <header>, <nav>, <main>, <article>, <aside>, <footer>, and <section> provide a logical document outline for screen readers.
  • Keyboard navigation: Gallery thumbnails have tabindex="0" and respond to Enter/Space key events. The modal close button receives focus on open.
  • ARIA attributes: The modal uses role="dialog", aria-modal="true", and aria-labelledby. Navigation uses aria-label.
  • Colour contrast: The chosen green/white combination meets WCAG AA contrast requirements. Text on the amber background uses dark green for adequate contrast.
  • Responsive design: The layout adapts to mobile viewports, ensuring usability across device sizes.
