2. Feedback & Program Directory

HTML Structure

The Feedback page uses the shared template from Student 1. The <main> section contains a scrollable program cards container, rating summary, reviews grid, and an interactive feedback form. I used <section> for the scrollable container and <fieldset> with <legend> to group form fields as required.

CSS Techniques

Page-specific CSS includes styling for program cards, scrollable container (max-height: 480px; overflow-y: auto), review cards, and form validation error states. Custom scrollbar styling was added for better UX. Flexbox and CSS Grid were used for responsive layouts.

JavaScript Implementation

  • Live character counter: Updates in real-time as user types in the message field, with colour warnings when approaching the 300-character limit.
  • Form validation: Checks required fields (name, email, programme, message) and validates email format. Error messages appear next to fields.
  • Submission confirmation: On successful validation, a confirmation message appears and the form resets.

Design Rationale

The scrollable program cards container allows users to browse multiple opportunities without overwhelming the page. The rating summary provides social proof, while review cards build trust. The form uses a clean, accessible layout with clear labels and real-time feedback to encourage submissions.
